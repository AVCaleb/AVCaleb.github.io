书籍电子化处理流水线

Book Digitization Pipeline using Qwen AI Models

概述 / Overview

本项目提供完整的书籍电子化处理流程，将PDF格式的古典书籍转换为：

  • YAML 数据文件：存储原文和中文翻译
  • HTML 网站：支持三种显示模式（原文、中译、对照）
  • PDF 文档：三个版本（原文版、中译版、中英对照版）

This pipeline converts classic books from PDF to:

  • YAML data files with original and translated content
  • HTML websites with three display modes
  • PDF documents in three versions

技术栈 / Tech Stack

  • AI Models (Qwen 系列):

    • qwen3-vl-plus: 视觉模型，用于 PDF 文字识别 (OCR)
    • qwen3-max: 语言模型，用于翻译

  • Python Libraries:

    • openai: API 调用
    • PyMuPDF (fitz): PDF 处理
    • PyYAML: 数据存储

  • PDF Generation:

    • XeLaTeX (需要安装 TeX Live 或 MacTeX)

HTML 显示模式 / HTML Display Modes

生成的 HTML 网站支持三种显示模式：

  1. English (原文): 仅显示原文
  2. 中文 (翻译): 仅显示中文翻译
  3. 对照 (Bilingual): 原文一段、中译一段并排显示

用户可通过页面顶部的按钮切换模式，偏好设置会保存在浏览器本地。

PDF 输出 / PDF Output

使用 XeLaTeX 生成三个版本的 PDF：

  1. *_en.pdf: 原文版
  2. *_cn.pdf: 中译版
  3. *_bilingual.pdf: 中英对照版

macOS 字体配置

默认使用以下字体：

  • 中文：PingFang SC
  • 英文：Times New Roman
  • 等宽：Menlo

可在 config.py 中修改字体设置。

圣经注释书特殊处理 / Bible Commentary Processing

对于圣经注释类书籍（如 Lightfoot 的歌罗西书注释），系统会：

  1. 识别页面顶部的希腊文/希伯来文经文
  2. 将分散在多页的经文合并
  3. 在 HTML 和 PDF 中以特殊样式显示经文块
  4. 保留原文的学术脚注

注意事项 / Notes

  1. API 配额：大型书籍处理可能消耗较多 API 配额
  2. 处理时间：完整流程可能需要较长时间，取决于书籍大小
  3. LaTeX 安装：PDF 生成需要完整的 LaTeX 环境
  4. 字体支持：确保系统安装了所需的中文字体
